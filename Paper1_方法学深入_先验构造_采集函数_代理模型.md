## 第一部分：物理约束 / 历史数据 / 领域知识 → 先验知识的形式化

核心思想：贝叶斯框架里"先验"有**四个可注入的入口**，领域知识按其数学性质对号入座，而不是一股脑塞进一个地方。下表是总览，后面逐条展开。

| 知识类型 | 数学性质 | 注入入口 | 形式化载体 |
|---|---|---|---|
| 经验关联式 / 叠加模型 | 对响应均值的趋势估计 | **GP 先验均值** \(m(x)\) | 参数化物理函数 \(m_{\text{phys}}(x;\theta)\) |
| 几何/物理对称性、周期性 | 函数的不变性结构 | **核函数** \(k(x,x')\) | 周期核、不变核、坐标变换 |
| 单调性 / 趋势方向（如 η 随 M 先增后减） | 导数符号约束 | **核函数 + 虚拟导数观测** | virtual derivative observations |
| 排间弱耦合 / 可分结构 | 函数的可加分解 | **核函数（结构）** | additive / 分组 ARD 核 |
| "最优解大概在哪"的工程判断 | 对 \(x^\star\) 位置的分布 | **采集函数 或 后验** | \(\pi(x)\)：πBO（采集）/ ColaBO（后验） |
| 硬物理约束（孔不重叠、冷气量上限） | 可行域边界 | **约束模型 / 参数化层** | 解析约束 + 可行性 GP |
| 量纲分析 / 无量纲化 | 输入空间的合理度规 | **输入变换** | 用 \(s/C, M, p/D, \alpha\) 等无量纲量作 GP 输入 |

### 1.1 经验关联式 → GP 先验均值（主通道，承载 80% 的领域知识）

这是你最熟悉、也最该重点写的通道。标准 BO 用零均值 GP，相当于"假设我对响应一无所知"；我们把关联式作为均值函数：

\[
f(x) \sim \mathcal{GP}\big(m_{\text{phys}}(x;\theta),\; k(x,x')\big), \qquad
f(x) = m_{\text{phys}}(x;\theta) + r(x), \quad r \sim \mathcal{GP}(0, k)
\]

GP 实际只对**残差** \(r(x) = f_{\text{CFD}}(x) - m_{\text{phys}}(x;\theta)\) 建模。残差来自关联式没刻画的物理（排间强耦合、曲率、压力梯度、转捩位置漂移），它比原函数光滑、量级小，少量样本即可拟合。

\(m_{\text{phys}}\) 的两层构造（与主计划 §4.1 一致，此处补充可学习性与"无损"性质）：

- 单排：\(\displaystyle \eta_{\text{row}}(s;M,D,p,\alpha)=\frac{c_0}{1+c_1\big[(s-s_{\text{hole}})/(M s_e)\big]^{c_2}}\)
- 多排叠加（Sellers）：\(\displaystyle \eta_{\text{total}}(s)=1-\prod_{i=1}^{N_{\text{row}}}\big(1-\eta_{\text{row},i}(s)\big)\)
- 面积分 → \(m_{\text{phys}}(x;\theta)=\bar\eta_{\text{corr}}(x)\)，\(\theta=(c_0,c_1,c_2)\)

**为什么这条通道天然"无损"（关键论证，回应你的 both 要求）：**
先验均值随数据累积被自动冲淡——在已观测点附近，GP 后验由数据主导；远离数据处才回落到 \(m_{\text{phys}}\)。当样本足够，后验与"零均值 GP + 足够数据"一致。因此：
- **先验对了** → 残差小、超参易学 → 早期大幅加速；
- **先验错了** → 残差大但 GP 仍能拟合，渐近性能不受损（不像 πBO 的衰减需要人为调 \(\beta\)）。
这就是"原则化注入（贝叶斯后验，非启发式）+ 保留收敛"两者兼得的方案。

**可学习先验**：把 \(\theta\) 注册为可训练参数，与核超参一起做 MAP：
\[
\hat\theta,\hat\phi=\arg\max_{\theta,\phi}\ \log p\big(\mathbf{y}\mid \mathbf{X},\theta,\phi\big)+\log p(\theta)
\]
\(p(\theta)\) 用以文献系数为中心的弱信息正态先验。好处：关联式在叶栅曲面上系数本就该偏离平板拟合值，让数据微调系数 = "数据校准的物理先验"，比固定系数稳健。**这是一个独立消融点。**

### 1.2 对称性 / 周期性 / 趋势 → 核函数

- **展向周期**：孔排沿展向周期布置，若把单周期内细节纳入，可在展向坐标用周期核 \(k_{\text{per}}(x,x')=\sigma^2\exp\!\big(-2\sin^2(\pi|x-x'|/p)/\ell^2\big)\)。本问题里展向多用周期边界，此项更多是"输入无量纲化"的延伸，列出备用。
- **单调/趋势先验**：若已知 \(\bar\eta\) 随某变量单调（小吹风比段效率随 M 增），可加**虚拟导数观测**（在若干点强加 \(\partial f/\partial x_j>0\) 的软观测），把趋势知识注入而不需真实样本。实现上 GPyTorch 可对导数 GP 联合建模。建议作为"轻量先验"消融项，不进主框架（避免趋势判断错误反噬）。

### 1.3 可分 / 弱耦合结构 → additive / 分组核（高维时的关键）

多排孔每排 4–5 参数，3–4 排即 15–20 维，逼近 GP-BO 舒适区上限。若排间耦合弱：
\[
f(x)\approx \sum_{i=1}^{N_{\text{row}}} f_i(x_{\text{row}_i}) \;\Rightarrow\; k(x,x')=\sum_i k_i(x_{\text{row}_i},x'_{\text{row}_i})
\]
可加核把有效维度从"全维"降到"每排维度"，大幅缓解维度灾难。注意：**排间耦合恰是本问题的物理核心**（也是 Paper 2 的卖点），所以不能假设完全可分。务实做法是"主效应可加核 + 低秩交互项"，或分组 ARD：让核自动学每排的相关长度，弱耦合维度长度尺度自然变大。维度上到 18+ 再考虑 SAASBO（稀疏轴对齐子空间先验）。

### 1.4 "最优解大概在哪" → 采集函数 / 后验（πBO vs ColaBO）

这是**对 \(x^\star\) 位置的先验** \(\pi(x)\)，与 1.1 的"对响应均值的先验"是不同对象，可叠加使用。两种注入法：

- **πBO（注入采集函数，启发式 + 有收敛保证）**：\(\alpha_{\pi}(x)=\alpha(x)\cdot\pi(x)^{\beta/n}\)，\(\beta/n\) 随迭代衰减。EI 下有 no-regret 证明，但先验是"人为乘进去再衰减"，信息会被主动丢弃。
- **ColaBO（注入后验，原则化）**：把 \(\pi(x)\) 作为 surrogate 上的额外先验，重塑 GP 后验，适配任意 MC 采集函数；更原则化但无已发表的形式 regret bound。

**本项目定位**：\(\pi(x)\) 可由关联式快速扫掠构造（\(\pi(x)\propto\exp(\gamma\,\bar\eta_{\text{corr}}(x))\)）。但由于我们已经用 1.1 把关联式作为均值注入，\(x^\star\)-先验与之**信息高度重叠**，不宜叠加过度。结论：**πBO 与 ColaBO 作为对比基线/消融，不作主方法**（主方法是先验均值 GP）。

### 1.5 硬约束 → 参数化层 + 可行性 GP

- **几何硬约束**（孔不重叠、不越尾缘、排序 \(s_1<s_2<s_3\)）：在参数化层用解析不等式直接保证，不进优化器。
- **物理/工程约束**（冷气量 \(\dot m_c\le\dot m_{c,\text{base}}\)、总压损失 \(\zeta\le\zeta_{\text{base}}(1+\epsilon)\)）：这些需 CFD 才知道，用约束模型。单目标主线下用 **constrained EI**：
\[
\alpha_{\text{cEI}}(x)=\mathbb{E}\big[\max(f(x)-f^\star,0)\big]\cdot\prod_j \Pr\big(c_j(x)\le 0\big)
\]
其中每个 \(c_j\) 用独立 GP 建模。网格失败的点作为**分类可行性 GP**（成功/失败二分类）单独建模，把"哪些设计算得出来"也学进去。

### 1.6 量纲分析 → 输入空间度规

把 GP 输入用无量纲量（\(s/C,\ M,\ p/D,\ \alpha,\ \beta\)）而非原始尺寸，等价于给输入空间一个物理合理的度规，让各向异性长度尺度更易学。这是"零成本但审稿人买账"的细节，写进 §Method 的 problem setup。

### 1.7 关于"历史数据"（按你的实际情况定调）

你确认手头**只有公开文献关联式、没有成体系可复用的历史 CFD/实验数据集**。因此：
- 领域知识全部走 1.1 关联式通道，**不引入迁移学习 / 多任务 GP / 元学习先验**（那类方法需要源任务数据集）；
- 文档/论文里明确这一点，避免审稿人问"为何不用 transfer learning"——答：无合规可复用源数据，且关联式先验已覆盖领域知识；
- **预留接口**：若后续课题组积累了别的叶型/工况的 CFD 库，可启用 multi-task GP（LMC / ICM）把它们作为额外保真或源任务。列入"未来工作"。

## 第二部分：采集函数对比与"无损改进"选择

### 2.1 候选采集函数评估（单目标，最大化 \(\bar\eta\)）

记 GP 后验均值 \(\mu(x)\)、标准差 \(\sigma(x)\)，当前最优 \(f^\star\)。

| 采集函数 | 形式 | 机理 | 优点 | 缺点 / 风险 | 适配本项目 |
|---|---|---|---|---|---|
| **PI**（改进概率） | \(\Phi\big(\frac{\mu-f^\star-\xi}{\sigma}\big)\) | 只问"是否更好" | 简单 | 过度利用、易陷局部 | ✗ 不推荐 |
| **EI**（期望改进） | \((\mu-f^\star-\xi)\Phi(z)+\sigma\phi(z)\) | 改进的期望幅度 | 解析、稳健、生态成熟 | 早期略偏利用；需调 \(\xi\) | ◯ 稳健基线 |
| **UCB** | \(\mu+\sqrt{\beta_t}\,\sigma\) | 乐观面对不确定 | 有理论 regret bound；探索可控 | \(\beta_t\) 需调度 | ◯ 理论对照基线 |
| **KG**（知识梯度） | \(\mathbb{E}[\max\mu^{(n+1)}]-\max\mu^{(n)}\) | 一步前瞻**全局**后验最优增益 | 可处理噪声/多保真/非"已评估点即解"场景 | 计算贵（嵌套 MC + 内层优化） | ★ 多保真主选 |
| **MES**（最大值熵搜索） | \(I(y;f^\star\!\mid\!x)\) | 减少对**最优值** \(f^\star\) 的熵 | 信息论"无损"、对尺度鲁棒、便于多保真 | 需采样 \(f^\star\)；实现略复杂 | ★ 信息论候选 |
| **JES**（联合熵搜索） | \(I(y;(x^\star,f^\star)\!\mid\!x)\) | 减少对**最优点+最优值**联合熵 | 比 MES 信息更全；SOTA 之一 | 计算最贵 | ◯ 进阶对照 |
| **qEI / qNEHVI** | 批量 / 多目标超体积 | 并行 / Pareto | 支持并行评估、多目标 | — | qEI 用于并行；qNEHVI 留多目标扩展 |

### 2.2 "无损"在这里的精确含义（三层，对应你的 both 要求）

"无损改进采集函数"可以从三个层面理解，本项目要求**同时满足前两层**：

1. **收敛无损（no-regret）**：注入先验后仍保证渐近收敛到真最优。
   - EI、UCB 本身有 regret bound；πBO 在 EI 上证明了"先验无论对错都保持 EI 的收敛速率"。
   - **先验均值 GP**：先验均值不改变 GP 的后验一致性（universal kernel + 先验均值仍是一致估计），渐近无损。✔
2. **先验注入无损（原则化、信息不被人为丢弃）**：先验以贝叶斯方式进入推断，而非乘一个随时间衰减的启发式因子。
   - πBO：✗（先验被 \(\beta/n\) 主动衰减，信息逐步丢弃）；
   - ColaBO / **先验均值 GP**：✔（先验进后验，由数据自然主导而非人为关闭）。
3. **信息论无损（采集层面）**：采集函数本身以"信息增益"为准则，不丢弃关于 \(f^\star\) 的信息——这是 MES/JES 的语义。

> **结论**：满足"both"的最干净方案是 **先验均值 GP（满足第 1、2 层）+ 信息论或 KG 采集函数（提供第 3 层）**。即把"先验注入"放到代理模型层（无损、原则化），把采集函数留给标准的、有理论保证的形式——**两个关注点解耦**，这也是论文方法叙述最清晰的写法。

### 2.3 最终采集函数选择（分场景）

- **若启用多保真（见第三部分判定）**：主采集函数用 **cost-aware MES / Multi-Fidelity MES**（或 MFKG），联合选点与选保真：
\[
(x^\star,\ell^\star)=\arg\max_{x,\ell}\ \frac{\alpha_{\text{info}}(x,\ell)}{\lambda_\ell}
\]
\(\alpha_{\text{info}}\) 为对 \(f^\star\)（最高保真）的信息增益，\(\lambda_\ell\) 为各保真实测成本。MES 比 KG 在多保真下更自然（信息增益对保真档可比、对输出尺度鲁棒），是主选；MFKG 作对照。
- **若单保真**：主用 **MES**（信息论无损），**EI / cEI** 作稳健基线与约束处理载体，**UCB** 作理论对照。
- **约束**：单目标主线统一用 constrained 形式（信息论采集亦可乘可行概率，或用 PESC/约束 MES）。
- **并行**：服务器可并行算多个 case 时，用批量版本（q-变体）一次推荐多个点，提升墙钟效率。

### 2.4 推荐的采集函数实验列（实验矩阵的一行维度）

主方法固定用 (MF-)MES；另起一组"采集函数消融"在便宜层级（第 0 级合成函数 / 第 1 级伪 CFD）上对比 EI / UCB / KG / MES / JES，证明选择合理。**真 CFD 上不重复跑全部采集函数**（太贵），只跑主选 + 1 个基线。

## 第三部分：是否引入多保真策略（判定，而非默认）

多保真不是免费午餐——它的收益**完全取决于低/高保真的相关性与成本比**，相关性差时反而拖累。给一个**先测后定**的判定流程（与主计划 §5.4 的先行诊断量挂钩）：

### 3.1 判定准则

在正式 BO 前，用第 1 级 LHS 样本里的 15–20 个点同时算 L1（粗 RANS）与 L2（细 RANS），计算：

1. **保真相关系数** \(\rho_{\text{L1,L2}}\)（Pearson，或秩相关 Spearman 应对非线性）；
2. **成本比** \(\lambda_{\text{L2}}/\lambda_{\text{L1}}\)（实测核时）；
3. **非线性度**：L1→L2 散点是否近线性（决定 AR(1) 还是 NARGP）。

判定表：

| 条件 | 决策 |
|---|---|
| \(\rho>0.8\) 且成本比 \(>5\) | **强上多保真**，AR(1) co-Kriging |
| \(0.6<\rho<0.8\)，成本比 \(>5\) | 上多保真，但用 **NARGP**（非线性自回归）防失配 |
| \(\rho<0.6\) | **放弃 L1↔L2 多保真**，退化为单保真 L2 + 关联式(L0)作先验均值 |
| 成本比 \(<3\) | 多保真收益不抵管理开销，**单保真即可** |

### 3.2 L0（关联式）始终保留为"零成本保真/先验"

无论 L1↔L2 是否成立，关联式 L0 都作为先验均值存在（第一部分 1.1）。所以本项目的"多保真"其实有两个独立决策：
- **L0 作先验均值**：✔ 始终启用（这是论文主贡献，几乎零成本零风险）；
- **L1↔L2 的 co-Kriging**：⚠ 由 §3.1 诊断决定，是"加分项"而非"必需项"。

这样设计的好处：**即使 L1↔L2 多保真被诊断否决，论文主线（物理先验 + BO 加速）依然完整成立**，多保真退为一个诚实报告的负结果/工况依赖结论。降低了项目风险。

### 3.3 多保真采集与代理（若启用）

- 代理：`SingleTaskMultiFidelityGP`（BoTorch，AR(1) 思想）或自定义 NARGP；
- 采集：MF-MES / MFKG（见 2.3）；
- 注意保真档之间**先验均值 \(m_{\text{phys}}\) 挂载位置**（挂 L1 层 vs 挂残差 \(\delta\)）是一个消融点（主计划 §4.2 已列）。

## 第四部分：代理模型选择（GP 为主，理由 + 变体 + 备选）

### 4.1 为什么 GP 是本问题的正确选择（你熟悉它，且它确实最优）

低样本、贵评估、需要不确定性量化的场景，GP 几乎是唯一合理选择：

| 模型 | 不确定性量化 | 小样本表现 | 注入先验均值 | 多保真支持 | 评价 |
|---|---|---|---|---|---|
| **GP** | 原生、校准好 | ★ 优 | 自然（mean module） | 成熟（co-Kriging/NARGP） | ★ 主选 |
| 随机森林（TPE/SMAC） | 粗糙、非校准 | 中 | 难 | 弱 | 高维离散场景才考虑 |
| 贝叶斯神经网 / DKL | 依赖近似 | 数据少时差 | 可但复杂 | 可 | 数据量大才有优势 |
| 深度 GP | 较好 | 训练不稳 | 复杂 | 可 | 强非线性/高维备选 |

结论：**标准 GP 作主代理**，与你的熟悉度一致，且生态（GPyTorch/BoTorch）对先验均值、多保真、约束、信息论采集全支持。

### 4.2 GP 的具体配置（写进论文的实现细节）

- **均值函数**：\(m_{\text{phys}}(x;\theta)\)（第一部分 1.1），\(\theta\) 可学习；
- **核函数**：**Matérn-5/2 + ARD**（各维独立长度尺度）作默认。Matérn-5/2 比 RBF 更适合工程响应（二阶可微，不过度光滑）；ARD 自动学每个设计变量的重要性，间接实现"维度筛选"；
- **噪声**：CFD 是确定性求解，但有离散/收敛残差噪声 → 用小的可学习同方差 noise（或固定一个小 jitter），别设成零（数值稳定性）；
- **输入变换**：无量纲化 + 归一化到单位立方体；**输出标准化**（残差均值 0、方差 1）利于超参优化；
- **超参推断**：MAP（带弱信息先验）起步；样本极少时可考虑对长度尺度做完全贝叶斯（NUTS 采样，BoTorch 的 `fully_bayesian`/SAAS），尤其维度高时；
- **高维（>15 维）退路**：SAASBO（稀疏轴对齐先验）或分组/可加核（第一部分 1.3）。

### 4.3 代理模型的验证（防止 GP 失配导致 BO 空转）

- 每隔若干样本做**留一交叉验证（LOO）**，看后验均值 vs 真值散点、标准化残差是否落在 ±2σ（校准性）；
- 监控长度尺度是否塌缩（过拟合）或爆炸（欠拟合）；
- 这套诊断与主计划 §5.4 "运行期保险机制"合并执行。

## 综合方法栈（一页纸总结）

```
设计向量 x（无量纲化输入：s/C, M, p/D, α, β × N_row）
        │
        ▼
┌─────────────────────────────────────────────┐
│ 代理模型：标准 GP                              │
│  · 先验均值 m_phys(x;θ)  ← 关联式+Sellers叠加  │ ← 领域知识主通道（无损、原则化注入）
│    （θ 可学习，数据自然主导，渐近无损）          │
│  · 核 Matérn-5/2 + ARD（高维→可加/SAAS）       │ ← 对称/可分结构
│  · 约束 GP（ζ, ṁc）+ 可行性分类 GP（网格失败）  │ ← 硬/软约束
│  · [可选] L1↔L2 co-Kriging / NARGP            │ ← 由 ρ 诊断决定是否启用
└─────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────┐
│ 采集函数：(MF-)MES 主选                         │
│  · 信息论无损；多保真下 cost-aware: α_info/λ_ℓ  │
│  · constrained 形式乘可行概率                   │
│  · EI/cEI 稳健基线，UCB 理论对照，KG/JES 进阶对照│
└─────────────────────────────────────────────┘
        │
        ▼   推荐 (x*, ℓ*) → CFD 评估 → 更新 GP（断点续跑、样本归档）
```

**一句话**：领域知识的"主力"经**先验均值**进代理模型（无损、原则化、数据自动校准）；采集函数选**信息论的 MES 族**（信息无损、天然适配多保真）；多保真**先诊断 ρ 再决定**是否启用 L1↔L2，而 L0 关联式作先验均值始终在场。这套组合同时满足你要的"原则化注入先验 + 保留收敛保证"。

## 仍需与你商量/确认的点（动笔正式论文前）

1. **MES 的实现成本**：信息论采集（MES/JES）实现比 EI 复杂，BoTorch 有 `qMaxValueEntropy` / `qMultiFidelityMaxValueEntropy` 现成实现。若你想降低工程量，可退而求其次用 **MFKG**（同样无损、多保真原生，API 更直接）作主采集。你倾向"理论最干净（MES）"还是"工程最省（MFKG）"？
2. **可学习先验系数 \(\theta\) 的自由度**：是只放开 \((c_0,c_1,c_2)\) 三个衰减系数，还是连 Sellers 叠加的加权也参数化？后者更灵活但更易过拟合小样本。建议先三系数，留作消融——除非你有更强的关联式形式想用。
3. **是否要把 πBO/ColaBO 真正实现为对比**：它们能强化"我们的先验注入更优"的论证，但增加约一个月工作量。若时间紧，可只实现 πBO（轻量、有现成参考）作为唯一的"采集层先验"对照，ColaBO 在 Related Work 里定性讨论。你的取舍？
4. **约束的优先级**：冷气量和总压损失，哪个是硬约束、哪个可放松进目标？这影响是用 constrained EI 还是把某项并入加权目标。主计划默认"冷气量硬约束、总压损失软约束"，确认是否如此。
